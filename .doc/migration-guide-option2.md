# 완전 무료(Free Tier) 아키텍처 마이그레이션 가이드

유지비 0원으로 24시간 돌아가는 AI 자동매매 시스템을 구축하기 위한 가이드입니다. 
(클라우드 DB: Supabase / 클라우드 서버: Google Cloud Platform)

---

## Step 1. Supabase (클라우드 DB) 세팅하기
가장 먼저 기존의 로컬 SQLite 파일을 대체할 클라우드 DB를 만듭니다.

1. **가입 및 프로젝트 생성**
   - [Supabase 공식 홈페이지](https://supabase.com/)에 접속하여 GitHub 계정으로 로그인합니다.
   - 대시보드에서 **`New Project`** 버튼을 클릭합니다.
   - 프로젝트 이름(예: `hanstock-ai`)과 DB 비밀번호를 설정합니다. (비밀번호는 반드시 기억해 두세요!)
   - **Region**은 `Seoul (South Korea)` 또는 `Tokyo`로 선택하고, 무료 요금제(Free)로 생성합니다. (몇 분 정도 소요됩니다.)

2. **접속 정보(URI) 복사하기**
   - 프로젝트 생성이 완료되면, 화면 우측 상단의 녹색 **`Connect`** 버튼을 클릭합니다.
   - 뜨는 팝업 창에서 상단 탭 중 **`URI`** 를 선택합니다.
   - ⚠️ **중요**: 팝업 창 하단의 **`Use connection pooling`** 체크박스를 반드시 **체크(ON)** 합니다.
   - `postgresql://postgres...` 로 시작하는 주소가 보이면 복사(Copy) 버튼을 누릅니다.
   - 복사한 주소에서 `[YOUR-PASSWORD]` 부분만 아까 1번에서 설정한 DB 비밀번호로 지우고 바꿔 적으시면 됩니다.

3. **시스템 환경변수(`.env`)에 등록하기**
   - 기존 코드의 `.env` 파일을 열고, 아래와 같이 복사한 URI를 추가해 줍니다.
     ```env
     DATABASE_URL="postgresql://postgres.프로젝트ID:내비밀번호@aws-0-ap-northeast-2.pooler.supabase.com:6543/postgres"
     ```

---

## Step 2. 구글 클라우드 GCP (24시간 무료 서버) 세팅하기
로컬 PC를 끄더라도 24시간 대시보드와 스케줄러가 돌아갈 수 있도록 가상 서버(VPS)를 만듭니다. 구글 클라우드는 특정 지역(미국)의 `e2-micro` 인스턴스를 평생 무료로 제공합니다.

1. **구글 클라우드 가입 및 콘솔 접속**
   - [Google Cloud Console](https://console.cloud.google.com/)에 접속하여 구글 계정으로 로그인합니다.
   - 무료 체험판 시작(결제 정보 등록)을 완료합니다. (무료 한도 내에서만 사용하면 결제되지 않습니다.)
   - 새 프로젝트를 하나 생성합니다. (예: `hanstock-auto`)

2. **Compute Engine 인스턴스 생성 (서버 만들기)**
   - 좌측 메뉴에서 **`Compute Engine` -> `VM 인스턴스`**로 이동하여 **[인스턴스 만들기]**를 클릭합니다.
   - 이름: `hanstock-server` 등 임의 지정
   - 🚨 **리전 및 영역 (매우 중요)**: 무료 혜택을 받으려면 반드시 아래 3곳 중 하나를 선택해야 합니다.
     - `us-west1 (오리건)`
     - `us-central1 (아이오와)`
     - `us-east1 (사우스캐롤라이나)`
   - 🚨 **머신 구성 (매우 중요)**: 
     - 시리즈: **`E2`** 선택
     - 머신 유형: **`e2-micro` (vCPU 2개, 1GB 메모리)** 선택
   - **부팅 디스크**: 
     - 운영체제: `Ubuntu`
     - 버전: `Ubuntu 24.04 LTS` 또는 `22.04 LTS`
     - 부팅 디스크 유형: **`표준 영구 디스크`** (크기는 30GB까지 무료)
   - **방화벽**: 
     - `HTTP 트래픽 허용`, `HTTPS 트래픽 허용` 모두 체크 (대시보드 웹 접속용)

3. **생성 및 접속 확인**
   - 맨 밑의 **[만들기]** 버튼을 클릭합니다.
   - 1~2분 뒤 초록색 체크 표시가 뜨면 서버가 생성된 것입니다!
   - 인스턴스 목록에 표시된 **`외부 IP`**를 복사해 둡니다.

4. **웹 브라우저로 바로 접속하기**
   - 인스턴스 목록의 `외부 IP` 옆에 있는 **`SSH`** 버튼을 클릭하면 별도의 프로그램 없이 웹 브라우저 안에서 검은색 터미널 창이 열리며 서버에 접속됩니다.

*(서버 생성이 완료되고 SSH 터미널 창이 열렸다면 알려주세요. 이후 터미널에 명령어를 입력하여 코드를 다운로드하고 서버를 24시간 가동시키는 세팅을 함께 진행하겠습니다!)*
